## Getting Started

The garage-operator is now running and watching for Garage custom resources.

### Create a Garage Cluster

Create a GarageCluster resource to deploy Garage:

  kubectl create secret generic garage-admin-token \
    --namespace default \
    --from-literal=admin-token=$(openssl rand -hex 32)

  cat <<EOF | kubectl apply -f -
  apiVersion: garage.rajsingh.info/v1beta2
  kind: GarageCluster
  metadata:
    name: my-garage
    namespace: default
  spec:
    zone: us-east-1
    replication:
      factor: 3
    storage:
      replicas: 3
      metadata:
        size: 10Gi
      data:
        size: 100Gi
    admin:
      adminTokenSecretRef:
        name: garage-admin-token
        key: admin-token
  EOF

### Create a Bucket

  cat <<EOF | kubectl apply -f -
  apiVersion: garage.rajsingh.info/v1beta1
  kind: GarageBucket
  metadata:
    name: my-bucket
    namespace: default
  spec:
    clusterRef:
      name: my-garage
  EOF

### Create an Access Key

  cat <<EOF | kubectl apply -f -
  apiVersion: garage.rajsingh.info/v1beta1
  kind: GarageKey
  metadata:
    name: my-key
    namespace: default
  spec:
    clusterRef:
      name: my-garage
    bucketPermissions:
      - bucketRef:
          name: my-bucket
        read: true
        write: true
  EOF

### Check Status

View all Garage resources:

  kubectl get garageclusters,garagebuckets,garagekeys,garagenodes

View operator logs:

  kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "garage-operator.name" . }} -f
